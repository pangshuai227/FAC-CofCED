# 基于 FaithfulRAG 思想改进 CofCED 的方案草稿

## 1. 任务理解

我们以 **CofCED** 作为 baseline。它来自论文 *A Coarse-to-fine Cascaded Evidence-Distillation Neural Network for Explainable Fake News Detection*，任务是：

> 给定一条 claim 和多篇相关 raw reports，预测 claim 的真实性标签，并从 raw reports 中抽取解释性证据句。

我们希望借鉴 **FaithfulRAG** 的思想。FaithfulRAG 的核心不是简单“用 RAG 回答问题”，而是：

> 当检索上下文和模型已有知识出现冲突时，先在事实粒度识别冲突，再进行证据对齐和推理融合。

因此，我们不建议直接把 CofCED 换成生成式 RAG 模型，而是建议把 FaithfulRAG 的 **fact-level conflict modeling** 融入 CofCED 的证据选择和真假分类流程。

## 2. Baseline CofCED 的核心流程

CofCED 包含三个子任务：

1. **Report Selection**
   - 从多篇 raw reports 中选择 top-K 篇 check-worthy reports。

2. **Sentence Selection**
   - 从选中的 reports 中抽取解释性证据句。

3. **Veracity Prediction**
   - 使用 claim、reports 表征、evidence 表征预测真实性标签。

原模型最终分类输入为：

```text
h_final = [h_claim; h_reports; h_evidence]
```

其中 `h_evidence` 是从被选中的证据句聚合得到的表示。

## 3. CofCED 的关键不足

CofCED 的句子选择器主要考虑四个因素：

```text
claim relevance + richness + salience - non-redundancy
```

也就是说，它更关心：

- 句子是否和 claim 相关；
- 句子信息是否丰富；
- 句子在原 report 中是否显著；
- 句子和已选证据是否重复。

但 fake news detection 的关键不只是“相关”，还包括：

- 这句话是在支持 claim，还是反驳 claim？
- 不同 raw reports 之间是否互相冲突？
- 是否存在大量重复转述 claim 但没有验证的句子？
- 是否有少数高质量证据推翻多数噪声报道？

因此，CofCED 的主要薄弱点可以概括为：

> 它能从噪声 reports 中抽取相关证据，但没有显式建模证据的支持、反驳和冲突关系。

## 4. FaithfulRAG 能提供什么启发

FaithfulRAG 有三个值得迁移的思想：

### 4.1 Self-Fact Mining

FaithfulRAG 先把问题相关的隐含知识外化为 self-facts。

迁移到 fake news detection 中，可以理解为：

> 将 claim 拆解为若干细粒度事实单元。

例如：

```text
Claim: Microwaving fabric masks is a good way to sanitize them for reuse.

Claim facts:
1. Fabric masks can be sanitized by microwaving.
2. Microwaving masks is safe.
3. The sanitized masks can be reused.
```

### 4.2 Contextual Knowledge Alignment

FaithfulRAG 将 self-facts 和 retrieved context chunks 对齐。

迁移到 CofCED 中，可以改为：

> 将 claim facts 与 raw reports 中的句子或事实单元进行对齐。

这样 evidence selection 不再只是句子级语义相似，而是事实级对齐。

### 4.3 Self-Think

FaithfulRAG 的 Self-Think 包括类似下面的步骤：

```text
Fact Analysis -> Matching -> Context Check -> Final Verification
```

迁移到 fake news detection 中，可以变成：

```text
Fact Analysis -> Label Hypothesis Matching -> Evidence Conflict Check -> Final Veracity Verification
```

这里不一定要让 LLM 直接生成最终判断，而是可以把这个过程转化为结构化特征或辅助模块。

## 5. 拟提出的方法

建议方法名称：

```text
FaC-CofCED: Fact-level Conflict-aware CofCED
```

中文可以叫：

> 事实级冲突感知的粗到细证据蒸馏模型。

核心思想：

> 在 CofCED 的 report selection 和 sentence selection 基础上，引入 claim-report 事实对齐、证据立场识别和跨报告冲突建模，使模型不仅能选择相关证据，还能区分支持证据、反驳证据和冲突证据。

## 6. 模型结构设计

### 6.1 Claim Fact Extraction

对 claim 进行事实拆解：

```text
F_c = {f_c1, f_c2, ..., f_cm}
```

实现方式有三种：

1. 轻量版：直接把 claim 当作一个整体 fact。
2. 中等版：用 LLM 或 OpenIE 离线抽取 claim facts。
3. 完整版：训练/微调 fact extraction 模块。

对于期末项目，建议至少做到轻量版；如果时间允许，做中等版。

### 6.2 Report Fact / Sentence Representation

对每个 report sentence `s_ij` 获得表示：

```text
h_ij = Encoder(s_ij)
```

如果做显式 fact extraction，也可以得到：

```text
F_sij = {f_ij1, f_ij2, ..., f_ijn}
```

但为了降低实现风险，可以先用 sentence 作为 fact proxy。

### 6.3 Fact-level Alignment Score

计算 claim facts 和 report sentence/facts 的对齐程度：

```text
A_ij = max cos(emb(f_c), emb(s_ij))
```

其中 `emb` 可以使用 FaithfulRAG 中类似的轻量句向量模型，例如：

```text
sentence-transformers/all-MiniLM-L6-v2
```

这个分数表示句子是否覆盖了 claim 的关键事实。

### 6.4 Stance / Conflict Score

使用 NLI 或 stance detection 模型判断句子与 claim 的关系：

```text
P_ij = [p_support, p_refute, p_neutral]
```

由此得到：

```text
support_score = p_support
refute_score = p_refute
conflict_score = p_refute 或 max conflict with other reports
```

对于假新闻检测，`refute_score` 很重要，因为很多关键解释证据本质上是在反驳 claim。

### 6.5 Cross-report Agreement

原 CofCED 有 non-redundancy，但没有 agreement。

我们可以新增跨报告一致性分数：

```text
AGR_ij = average agreement between s_ij and evidence from other reports
```

直觉是：

- 如果多个不同 reports 都反驳同一个 claim fact，这些证据更可靠；
- 如果某个句子孤立地支持 claim，但多数证据反驳 claim，它可能是噪声或谣言传播。

## 7. 改进后的 Sentence Selector

原 CofCED 的 sentence selector 可以抽象为：

```text
score(s_ij)
= relevance
+ richness
+ salience
- redundancy
```

我们扩展为：

```text
score(s_ij)
= relevance
+ richness
+ salience
- redundancy
+ fact_alignment
+ stance_salience
+ cross_report_agreement
```

这样选出的 evidence 不只是“看起来相关”，而是更可能对真假判断有判别力。

## 8. 改进后的 Veracity Prediction

原 CofCED 将所有 evidence 聚合为一个 `h_evidence`。

我们建议拆成三类：

```text
h_support  = Pool(supporting evidence)
h_refute   = Pool(refuting evidence)
h_conflict = Pool(conflicting evidence)
```

最终分类输入变为：

```text
h_final = [h_claim; h_reports; h_support; h_refute; h_conflict; h_agreement]
```

这会比单一 `h_evidence` 更适合 fake news detection，因为真实性判断往往取决于证据方向，而不只是证据数量。

## 9. 训练目标

保留 CofCED 原本的多任务损失：

```text
L_all = beta_D * L_report
      + beta_S * L_sentence
      + beta_C * L_classification
```

可选增加 stance 辅助损失：

```text
L_all = beta_D * L_report
      + beta_S * L_sentence
      + beta_C * L_classification
      + beta_T * L_stance
```

其中 stance label 可以来自：

- 现成 NLI 模型生成的伪标签；
- 少量人工标注；
- 规则或 weak supervision。

如果时间紧，stance 不一定作为 loss，可以只作为额外 feature 输入模型。

## 10. 实验设计

### 10.1 主实验

沿用 CofCED 的两个数据集：

- RAWFC
- LIAR-RAW

主要指标：

- Precision
- Recall
- macro-F1

解释质量指标：

- ROUGE-1
- ROUGE-2
- ROUGE-L

### 10.2 消融实验

建议实验表：

| Model | FA | ST | AGR | CER | macro-F1 |
| --- | --- | --- | --- | --- | --- |
| CofCED | - | - | - | - |  |
| + Fact Alignment | yes | - | - | - |  |
| + Stance Score | yes | yes | - | - |  |
| + Agreement | yes | yes | yes | - |  |
| Full FaC-CofCED | yes | yes | yes | yes |  |

其中：

- FA = Fact Alignment
- ST = Stance / Conflict Score
- AGR = Cross-report Agreement
- CER = Conflict-aware Evidence Representation

### 10.3 解释效果分析

除了 ROUGE，还可以做 case study：

展示 claim、原 CofCED 选出的证据、FaC-CofCED 选出的证据。

重点观察：

- 是否选到了反驳 claim 的关键句；
- 是否避免了仅仅复述 claim 的句子；
- 是否能显示多个 reports 的一致反驳；
- 是否能解释为什么预测为 false / half / true。

## 11. 推荐实现路线

如果时间有限，建议按下面顺序做：

### Step 1: 复现 CofCED

先获得原始代码和数据，跑通 baseline。

### Step 2: 加入 NLI stance 特征

对每个 claim-sentence pair 计算：

```text
[p_entailment, p_contradiction, p_neutral]
```

映射为：

```text
support / refute / neutral
```

### Step 3: 加入 alignment 特征

使用 sentence-transformers 计算 claim 与 sentence 的 embedding similarity。

### Step 4: 修改 evidence pooling

将 evidence 分成 support/refute/conflict 三组聚合。

### Step 5: 做消融和 case study

证明新增模块确实提升了分类或解释质量。

## 12. 方案价值

这个方案的优点是：

1. **贴合 baseline**
   - 没有推翻 CofCED，而是在其最核心的 evidence distillation 上增强。

2. **贴合 FaithfulRAG**
   - 借鉴的是 fact-level alignment 和 conflict modeling，而不是表面上的 RAG 生成形式。

3. **适合 fake news detection**
   - 假新闻检测天然需要处理支持、反驳和冲突证据。

4. **容易做消融**
   - 每个新增模块都能单独去掉，实验叙事清晰。

5. **实现风险可控**
   - 可以先用 NLI 和 sentence-transformers 做轻量版，再逐步加入 LLM fact extraction。

## 13. 一句话总结

我们的方法可以概括为：

> Inspired by FaithfulRAG, we enhance CofCED with fact-level alignment and conflict-aware evidence modeling, enabling the model to distinguish supporting, refuting, and conflicting evidence from noisy raw reports for more accurate and explainable fake news detection.

中文表述：

> 受 FaithfulRAG 启发，我们在 CofCED 中引入事实级对齐和冲突感知证据建模，使模型能够从噪声 raw reports 中区分支持、反驳和冲突证据，从而提升可解释假新闻检测的准确性和解释质量。
